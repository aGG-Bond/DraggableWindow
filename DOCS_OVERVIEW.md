## 📋 文档更新规范

### CHANGELOG.md 更新时机

- ✅ 发布新版本时
- ✅ 添加新功能时
- ✅ 修复重要 Bug 时
- ✅ 进行破坏性变更时

### 更新内容分类

#### Added (新增)
- 新功能
- 新特性
- 新配置选项

#### Changed (变更)
- 现有功能的改进
- API 的变化
- 行为的调整

#### Deprecated (废弃)
- 即将移除的功能
- 不推荐使用的 API

#### Removed (移除)
- 已删除的功能
- 不再支持的 API

#### Fixed (修复)
- Bug 修复
- 性能优化
- 安全问题修复

#### Security (安全)
- 安全相关的更新
- 漏洞修复

## 🎯 最佳实践

### 编写更新日志的技巧

1. **简洁明了**: 用简短的语言描述变更
2. **分类清晰**: 使用标准分类（Added/Changed/Fixed 等）
3. **提供示例**: 对于重大变更，提供代码示例
4. **注明影响**: 说明变更对现有代码的影响
5. **链接相关**: 添加相关的 Issue/PR 链接

### 不好的例子 ❌

```markdown
- 修复了一些问题
- 改进了性能
- 更新了依赖
```

### 好的例子 ✅

```markdown
### Fixed
- 修复了容器模式下 padding 配置不生效的问题 (#42)
- 修复了移动端触摸事件在 iOS Safari 上的兼容性问题

### Changed
- 将拖拽阈值从 5px 调整为 10px，减少误触发
- 更新了 Rollup 到 v3.29.0，提升构建速度

### Added
- 添加了 TypeScript 类型定义文件
- 添加了 onDragStart/onDragEnd 回调函数支持
```
